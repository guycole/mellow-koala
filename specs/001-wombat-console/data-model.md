## Entity Details

### Task

**Purpose**: Represents a single execution of a named job from a Mellow project.

| Column | Type | Nullable | Constraints | Notes |
|--------|------|----------|-------------|-------|
| `id` | bigint | NO | PK | Auto-generated |
| `name` | string | NO | NOT NULL | Job name (e.g., `"weather_import"`) |
| `uuid` | string | NO | NOT NULL, UNIQUE | Natural dedup key for imports |
| `host` | string | NO | NOT NULL | Machine that ran the task |
| `start_time` | datetime | NO | NOT NULL | Task start timestamp |
| `stop_time` | datetime | YES | — | Null = task still running |
| `created_at` | datetime | NO | — | Rails timestamp |
| `updated_at` | datetime | NO | — | Rails timestamp |

**Derived attributes** (never stored):
- `duration` → `Float` seconds (`stop_time - start_time`), or `nil` if `stop_time` is null
- `duration_display` → `"HH:MM:SS"` string, or `"In Progress"` if `stop_time` is null

**Indexes**:
- `UNIQUE INDEX tasks_uuid` on `uuid` (required for `upsert_all unique_by: :uuid`)
- `INDEX tasks_host` on `host` (filter by host — User Story 1 acceptance scenario 3)
- `INDEX tasks_start_time` on `start_time` (filter/sort by date range — scenario 4)

**Validations** (model layer):
- `uuid` — presence, uniqueness (belt-and-suspenders with DB constraint)
- `name`, `host`, `start_time` — presence

**State transitions**:
- Created with `stop_time: nil` → task "In Progress"
- Import sets `stop_time` value → task "Complete"
- No state machine required; state is derived from `stop_time` nullness

### BoxScore

**Purpose**: Aggregated metric snapshot for a task run from a Mellow project.

| Column | Type | Nullable | Constraints | Notes |
|--------|------|----------|-------------|-------|
| `id` | bigint | NO | PK | Auto-generated |
| `task_name` | string | NO | NOT NULL | Name of the associated task |
| `task_uuid` | string | NO | NOT NULL | UUID of the associated task (informational, no FK) |
| `population` | decimal(10,4) | NO | NOT NULL | Metric value; decimal to support fractional values |
| `time_stamp` | datetime | NO | NOT NULL | When the metric was recorded |
| `uuid` | string | NO | NOT NULL, UNIQUE | Natural dedup key for imports |
| `created_at` | datetime | NO | — | Rails timestamp |
| `updated_at` | datetime | NO | — | Rails timestamp |

**Indexes**:
- `UNIQUE INDEX box_scores_uuid` on `uuid` (required for `upsert_all`)
- `INDEX box_scores_task_name` on `task_name` (filter by task name — User Story 2)
- `INDEX box_scores_id` on `id` (Pagy keyset pagination requires monotonic order column)

**Validations**:
- `uuid`, `task_name`, `task_uuid`, `time_stamp` — presence
- `population` — presence, numericality (greater than or equal to 0)

**Notes**:
- No foreign key to `tasks.uuid` at the database level for v1 (spec Assumption §5).
  `task_uuid` is stored as a plain string for informational cross-reference only.
- `population` uses `decimal(10,4)` to accommodate both integer and fractional values
  without requiring a future migration (research R-005).

### ImportLog

**Purpose**: Audit record for a single Rake task import run (FR-006, FR-016).

| Column | Type | Nullable | Constraints | Notes |
|--------|------|----------|-------------|-------|
| `id` | bigint | NO | PK | |
| `source_file` | string | NO | NOT NULL | File basename (e.g., `"tasks_20240101.json"`) |
| `import_type` | string | NO | NOT NULL | `"tasks"` or `"box_scores"` |
| `run_at` | datetime | NO | NOT NULL | When the import ran (`Time.current` at start) |
| `records_processed` | integer | NO | NOT NULL | Total records in the JSON file |
| `records_inserted` | integer | NO | NOT NULL | New records added to DB |
| `records_skipped` | integer | NO | NOT NULL | Duplicates (by uuid) skipped |
| `error_details` | text | YES | — | Newline-separated per-record errors; null if none |
| `created_at` | datetime | NO | — | |
| `updated_at` | datetime | NO | — | |

**Indexes**:
- `INDEX import_logs_run_at` on `run_at DESC` (audit log viewer shows newest first)

**Validations**:
- `source_file`, `import_type`, `run_at` — presence
- `records_processed`, `records_inserted`, `records_skipped` — presence, numericality ≥ 0
- `import_type` — inclusion in `%w[tasks box_scores]`

| Column | Type | Nullable | Notes |

| `user_id` | integer | NO | FK → `users.id` |
| `ip_address` | string | YES | Recorded at login |
| `user_agent` | text | YES | Recorded at login |
| `touched_at` | datetime | NO | Updated on each request; used for inactivity timeout (FR-010) |
| `created_at` | datetime | NO | |
| `updated_at` | datetime | NO | |

**Session timeout**: `SESSION_TIMEOUT = ENV.fetch('SESSION_TIMEOUT_HOURS', '8').to_i.hours`.

## Edge Case Handling

| Scenario | Handling |
|----------|----------|
| `stop_time` is null | `duration_display` returns `"In Progress"` (FR-013) |
| JSON import with duplicate UUID | `upsert_all unique_by: :uuid` skips; counted in `records_skipped` |
| Concurrent imports of same file | DB unique constraint prevents duplicates; both imports complete cleanly |
| UUID exists but different field values | Existing record preserved (upsert updates to same values); counted as skipped |
| Empty JSON array `[]` | Processes 0 records; audit log records `processed: 0`; exits cleanly |
| Malformed JSON record in file | Caught per-record in `handle_record`; logged to `error_details`; import continues |
| Non-existent file path | `ImportService` raises `ArgumentError`; Rake task prints error and `exit 1` |
| Pagination beyond last page | Pagy returns empty collection; no 500 error |
| `population` with fractional value | `decimal(10,4)` stores correctly without precision loss |
